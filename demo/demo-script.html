What iAgentX is

The Challenge

Why AI can't help IBM i developers — yet

"The real issue is awareness. Copilot can actually call Code for IBM i extension commands — but only if you explicitly tell it those tools exist. Without that context, it won't go looking. Claude Code and IBM Bob have no IBM i access at all. And the one vendor adding real integration — IBM Bob — is making it a paid feature launching June 24th. iAgentX solves this by automatically giving the AI full context of every available IBM i tool — no prompting required."

💡 Pause on "only if you tell it" — this is the nuance that surprises developers who tried Copilot and assumed it worked automatically.

Introducing iAgentX

The solution

"iAgentX is a VS Code extension. Install it — that's it. It auto-configures Claude Code, Copilot, and IBM Bob on first launch. No manual MCP setup. And crucially, it reuses the SSH connection you already have from Code for IBM i — no new credentials, no extra setup."

💡 Emphasise "reuses your connection" — this is the biggest friction-removal point for enterprise teams with strict credential policies.
~30 sec
05

Architecture

How it all connects

"Under the hood it's a lightweight MCP server running inside the VS Code extension host, bound only to localhost. AI agents call it over HTTP — it forwards requests through Code for IBM i's existing SSH session. Nothing is exposed to the network. One server instance serves all your VS Code windows."

💡 Point to the diagram layers as you speak. You don't need to explain every layer — just trace the path: AI → iAgentX → SSH → IBM i.
~35 sec
06

Security

Built-in, not bolted-on

"Security was designed in from day one. The AI runs under your own IBM i user profile — no service accounts, no elevated privileges. SQL is strictly read-only — SELECT, WITH, VALUES only — enforced at the extension layer, not just a convention. CL commands are allowlisted to display and diagnostic verbs only. And editor writes only touch files you already have open."

💡 This slide often generates questions in enterprise settings. Be ready for "what if someone modifies the allowlist?" — answer: it's open source, auditable, and enforced server-side.
~40 sec

Demo 1 — Row Count Analysis

Cross-table row count across a full schema

"Let's see it in action. I ask Claude Code: 'Get row counts for all main tables and their history tables in ECMDTZDTA.' It calls ibmi_run_sql, queries the system catalog, dynamically builds the COUNT queries, and returns a clean result — 18 table pairs in seconds. Manually writing that SQL would take most developers 20 minutes."

💡 If running live — paste the prompt and let it run in real time. The tool call appearing in chat is the "wow moment." Point it out explicitly.

Demo 2 — Fix FK Hierarchy Function

Debug & fix a SQL table function

"Second demo — a real debugging scenario. A SQL table function is returning level 1 for every table, ignoring all the FK relationships. I ask Claude Code to investigate. It reads the open source member directly from my editor using ibmi_get_active_editor — no copy-paste — then queries the FK catalog. Root cause: wrong column name in the recursive CTE join. It finds it, fixes it, and gives me the CREATE statement ready to deploy to library DK. That's the key point — the AI saw my open file without me doing anything."

💡 Stress "without me doing anything" — the audience should feel the friction that just disappeared. This is the most technically impressive moment of the demo.
